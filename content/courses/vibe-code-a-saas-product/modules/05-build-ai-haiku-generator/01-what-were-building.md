# What We're Building

Our second project is an AI-powered haiku generator. It sounds simple — and it is — but this project has a twist that teaches one of the most important security lessons in modern software.

## The App

A single-page app where you:

- Type in a topic (like "sunset" or "coffee" or "debugging")
- Click a button
- Get back a haiku generated by an AI model

Clean interface. One input, one button, one output. Maybe a small gallery of recently generated haikus.

## Why This Project?

Three reasons:

**1. You'll integrate an AI API.** This is the same pattern you'd use to add AI features to your SaaS — summarization, content generation, chatbots, recommendations. If your product has any AI component, this is the foundation.

**2. You'll learn about prompt injection.** We're going to purposely build this app with a security flaw. Then we'll exploit it. Then we'll fix it. This is hands-down the most important security concept for anyone building AI-powered applications, and I want you to experience it firsthand.

**3. You'll practice the workflow.** Scaffold, build, commit, push, deploy. Same rhythm as the image uploader, but now it's becoming second nature.

## The Security Angle

Here's the thing that most tutorials won't teach you: **user input is dangerous.** Every field where a user can type text is a potential attack vector. In traditional web apps, we worry about XSS (Cross-Site Scripting) and SQL injection. In AI-powered apps, we worry about prompt injection.

We're going to build the haiku generator in a way that's vulnerable to prompt injection on purpose. You'll see how a clever user can manipulate the AI into doing things it shouldn't. Then you'll understand *why* input validation matters — not because a textbook says so, but because you saw the exploit with your own eyes.
